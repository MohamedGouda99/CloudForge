# Implementation Plan: AWS Resource Classification Audit

**Branch**: `006-aws-resource-classification` | **Date**: 2026-01-15 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/006-aws-resource-classification/spec.md`

## Summary

Audit all AWS services across 12 categories to classify each resource as either "container" (can hold child resources visually) or "icon" (standalone). Update the shared/resource-catalog package with correct classifications and containmentRules, replace hardcoded containment logic in projects.py with dynamic catalog lookups, and ensure the frontend React Flow canvas correctly renders container vs icon nodes.

## Technical Context

**Language/Version**: Python 3.11 (backend), TypeScript 5.6 (frontend, shared catalog)
**Primary Dependencies**: FastAPI, SQLAlchemy 2.0, React 18, React Flow, Zustand
**Storage**: PostgreSQL 15, localStorage (frontend cache)
**Testing**: pytest (backend), vitest (frontend/shared)
**Target Platform**: Web application (Docker-based deployment)
**Project Type**: Web (frontend + backend + shared package)
**Performance Goals**: Canvas 60 FPS with 100 nodes, catalog API < 200ms p95
**Constraints**: Cached fallback for offline, warning indicators for invalid containment
**Scale/Scope**: ~150 AWS resource types across 12 categories
